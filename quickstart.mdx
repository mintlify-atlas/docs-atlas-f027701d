---
title: "Quickstart"
description: "Get cineCDMX running locally in under 10 minutes."
---

This guide walks you through cloning the repository, configuring your environment, and running cineCDMX on your local machine.

<Note>
  You need Python 3.10+, PostgreSQL, and a [free TMDB API key](https://www.themoviedb.org/settings/api) before starting. Google OAuth is optional for local development.
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/IsaacMO214/cineCDMX.git
    cd cineCDMX
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    <CodeGroup>
      ```bash macOS / Linux
      python3 -m venv venv
      source venv/bin/activate
      ```

      ```bash Windows
      python -m venv venv
      venv\Scripts\activate
      ```
    </CodeGroup>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    Key packages installed:
    - **Django 5.2** — web framework
    - **djangorestframework** — REST API support
    - **django-allauth** — Google OAuth
    - **psycopg2-binary** — PostgreSQL adapter
    - **gunicorn** — production WSGI server
    - **whitenoise** — static file serving
    - **python-dotenv** — `.env` file loading
    - **dj-database-url** — parses `DATABASE_URL`
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash .env
    SECRET_KEY=your-secret-key-here
    DEBUG=True
    DATABASE_URL=postgresql://user:password@localhost:5432/cinecdmx
    ALLOWED_HOSTS=127.0.0.1,localhost
    TMDB_API_KEY=your-tmdb-api-key-here

    # Optional — only needed for Google OAuth
    GOOGLE_CLIENT_ID=
    GOOGLE_CLIENT_SECRET=
    ```

    | Variable | Required | Description |
    |----------|----------|-------------|
    | `SECRET_KEY` | Yes | Django secret key — use a strong random value |
    | `DEBUG` | No | Set to `True` for development (default: `False`) |
    | `DATABASE_URL` | Yes | PostgreSQL connection URL |
    | `ALLOWED_HOSTS` | No | Comma-separated hostnames (default: `127.0.0.1,localhost`) |
    | `TMDB_API_KEY` | Yes* | Required for movie poster and metadata features |
    | `GOOGLE_CLIENT_ID` | No | Required for Google OAuth sign-in |
    | `GOOGLE_CLIENT_SECRET` | No | Required for Google OAuth sign-in |

    <Tip>
      Get a free TMDB API key at [themoviedb.org/settings/api](https://www.themoviedb.org/settings/api). It's free and instant.
    </Tip>
  </Step>

  <Step title="Run database migrations">
    Make sure PostgreSQL is running and the database exists, then apply all migrations:

    ```bash
    python manage.py migrate
    ```

    This creates all tables: `Sucursal`, `Sala`, `Genero`, `Pelicula`, `Funcion`, `Boleto`, `Usuario`, plus all allauth and session tables.
  </Step>

  <Step title="Seed sample data (optional)">
    Populate the database with sample branches, movies, and screenings:

    ```bash
    python poblar_bd.py
    ```

    This script creates:
    - **4 branches**: CDMX Norte, CDMX Oriente, Reforma, Perisur
    - **2 salas**: Sala 1 (CDMX Norte), Sala 2 (Perisur)
    - **10 genres**: Acción/Ciencia Ficción, Suspense/Misterio, Comedia, Terror, Drama, Aventura, Animación, Romance, Fantasía, Documental
    - **2 movies**: Guardianes de la Galaxia (TMDB ID: 118340, $80.00), Scream 7 (TMDB ID: 1159559, $85.00)
    - **3 funciones** scheduled for today

    <Note>
      The seed script inserts records directly via the ORM, bypassing `PeliculaForm` validation. The employee form restricts `costo_entrada` to $32.00 (Común) or $80.00 (Estreno), but seeded data can use other values like $85.00.
    </Note>

    <Warning>
      `poblar_bd.py` deletes all existing data before seeding. Do not run it against a database with real data.
    </Warning>
  </Step>

  <Step title="Create a superuser">
    Create a Django admin superuser to access `/admin/`:

    ```bash
    python manage.py createsuperuser
    ```

    <Note>
      To create an **Admin** role user for the cineCDMX interface (not Django admin), log in to the Django admin at `/admin/` and create a `Usuario` record with `rol=ADMIN`. There is no self-registration in cineCDMX — all accounts are created by an Admin via `/usuarios/` or directly in the database.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    python manage.py runserver
    ```
  </Step>

  <Step title="Open the app">
    Visit [http://localhost:8000](http://localhost:8000) to see the cartelera (movie listings).

    | URL | Description |
    |-----|-------------|
    | `http://localhost:8000/` | Public cartelera |
    | `http://localhost:8000/login/` | Login page |
    | `http://localhost:8000/dashboard/` | Employee dashboard |
    | `http://localhost:8000/admin/` | Django admin |
  </Step>
</Steps>
